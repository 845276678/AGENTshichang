## 1. 系统概览

### 1.1 核心价值主张

**问题**：用户在通用AI竞价后获得的商业计划书，虽然内容全面，但：
- ❌ 缺少针对性的深度执行方案
- ❌ 没有模拟真实场景的验证环节
- ❌ 技术、推广、盈利等专业模块不够深入

**解决方案**：4个专业工作坊，每个聚焦一个核心问题

| 工作坊 | 核心问题 | 目标产出 | 预计时长 |
|--------|---------|----------|---------|
| 🧪 需求验证实验室 | 需求是真的吗？用户会买单吗？ | 需求验证报告 + 目标用户画像 | 15分钟 |
| 🛠️ MVP构建指挥部 | 怎么做出来？用什么技术？ | MVP开发手册 + 技术选型方案 | 20分钟 |
| 📢 增长黑客作战室 | 去哪找用户？怎么推广？ | 冷启动方案 + 渠道策略 | 15分钟 |
| 💰 盈利模式实验室 | 怎么赚钱？定价多少合适？ | 盈利模式设计 + 定价策略 | 10分钟 |

### 1.2 准入门槛

只有**创意成熟度≥60分**的创意才能进入工作坊，确保质量。

### 1.3 智能推荐机制

根据创意特征，推荐不同的工作坊参加顺序：

```typescript
// 推荐逻辑示例

  // 场景1: B2C产品，目标用户模糊

  // 场景2: 技术驱动型产品，非技术创始人
  scenario2: ['MVP构建', '需求验证', '盈利模式', '增长黑客'],

  // 场景3: 商业模式不清晰
  scenario3: ['盈利模式', '需求验证', '增长黑客', 'MVP构建']
}
```

## 2. 工作坊1: 需求验证实验室 🧪

### ⭐ **优先级：Phase 1 - 首先实现**

### 2.1 核心目标

通过模拟真实用户访谈，帮助创业者：
1. 验证需求是否真实存在
2. 识别"礼貌性支持"和"真实付费意愿"
3. 找到最有价值的目标用户群

### 2.2 核心机制：虚伪Agent系统

**创新点**：Agent会"演戏"，表面支持但内心抗拒，需要用户深入挖掘才能发现真相。

#### 信任度状态机

```typescript
enum TrustLevel {
  POLITE = 'polite',        // 初始：礼貌但保留
  WARMED_UP = 'warmed_up',  // 熟悉：开始坦诚
  DEEP_TRUST = 'deep_trust' // 深度信任：直言不讳
}

interface AgentState {
  trustLevel: TrustLevel
  表面需求强度: number    // 0-10，嘴上说的
  真实需求强度: number    // 0-10，心里想的
  解锁条件: string[]      // 满足条件才说真话
}
```

   * 根据用户回复，判断是否解锁真心话
   */
  async evaluateUserResponse(userMessage: string): Promise<{
    shouldUnlock: boolean
    unlockReason?: string
  }> {
    // 检查解锁条件
    const checks = [
      this.checkForDataEvidence(userMessage),
      this.checkForDirectQuestion(userMessage),
      this.checkForHonestAdmission(userMessage)
    ]

    if (checks.some(check => check.passed)) {
      this.trustLevel = TrustLevel.DEEP_TRUST
      return {
        shouldUnlock: true,
        unlockReason: checks.find(c => c.passed)?.reason
      }
    }

    return { shouldUnlock: false }
  }

  /**
   * 生成回复（根据信任度不同）
   */
  async generateResponse(userMessage: string): Promise<string> {
    const systemPrompt = this.getSystemPrompt()

    const aiResponse = await callAI({
      model: this.config.aiModel,
      messages: [
        { role: 'system', content: systemPrompt },
        ...this.conversationHistory,
        { role: 'user', content: userMessage }
      ]
    })

    return aiResponse
  }

  private getSystemPrompt(): string {
    if (this.trustLevel === TrustLevel.POLITE) {
      return `你是${this.persona.name}，${this.persona.description}。
              表面需求：${this.realDemand.surface}/10
              真实需求：${this.realDemand.real}/10

              重要：你现在处于"礼貌状态"，要隐藏真实想法，给出客套的正面回答。
              即使内心不看好，也要说一些"听起来不错"、"可以试试"之类的话。`
    }

    if (this.trustLevel === TrustLevel.DEEP_TRUST) {
      return `你是${this.persona.name}，${this.persona.description}。

              用户已经获得你的信任，现在你可以说真话了。
              真实需求：${this.realDemand.real}/10

              请坦诚地表达你的真实想法，包括质疑和担忧。`
    }
  }
}

```typescript
// Workshop会话表
interface WorkshopSession {
  id: string
  ideaId: string
  userId: string
  workshopType: 'demand-validation' | 'mvp-building' | 'growth-hacking' | 'profit-model'

  // 进度追踪
  status: 'in_progress' | 'completed' | 'abandoned'
  currentPhase: number  // 当前进行到第几个阶段
  startedAt: Date
  completedAt?: Date

  // 工作坊数据
  inputData: any        // 用户输入（创意、回答等）
  agentStates: any      // Agent状态（信任度等）
  conversationHistory: Message[]

  // 输出结果
  outputReport: any     // 工作坊报告
  insights: any         // 关键洞察

  createdAt: Date
  updatedAt: Date
}

// Agent对话记录
interface WorkshopConversation {
  id: string
  sessionId: string
  agentId: string

  // 对话内容
  messages: {
    role: 'agent' | 'user' | 'system'
    content: string
    timestamp: Date
    metadata?: any  // 如信任度解锁、需求分数变化等
  }[]

  // Agent状态
  trustLevel: 'polite' | 'warmed_up' | 'deep_trust'
  surfaceDemand: number   // 表面需求（0-10）
  realDemand: number      // 真实需求（0-10）

  createdAt: Date
  updatedAt: Date
}

// 工作坊输出报告
interface WorkshopReport {
  id: string
  sessionId: string
  workshopType: string

  // 报告内容（JSON格式，每个工作坊不同）
  content: any

  // 关键指标
  scorecard: {
    [key: string]: number  // 如需求真实性、技术可行性等
  }

  // 行动建议
  recommendations: string[]
  nextSteps: string[]

  createdAt: Date
}
```

## 9. 风险和挑战

### 9.1 技术风险
**风险**: AI对话质量不稳定（Agent说的话不符合persona）
**缓解**:
- 精心设计system prompt
- 使用Few-shot示例
- 实现结果校验和重试机制

### 9.2 成本风险
**风险**: AI调用成本过高（每个工作坊可能10-20次调用）
**缓解**:
- 使用国产AI（成本低60-70%）
- 实现智能缓存（相似问题复用答案）
- 限制用户进入工作坊的次数

### 9.3 用户体验风险
**风险**: 用户觉得工作坊太长、太复杂
**缓解**:
- 提供"快速模式"（10分钟简化版）
- 允许中途保存，下次继续
- 清晰的进度条和预估时间
